What is this?

"PrivacyBook" is a mock social network login workflow designed to demonstrate one possible implementation of the Poze user verification API. In this case, the site represents an entirely fictional social network that used to be exceptionally popular (boasting billions of regular users), but recently has faced an overwhelming problem with fake "bot accounts" masqurading as real people. One account, belonging to username conlonn@andrew.cmu.edu, has been (wrongly) flagged as a possible "bot account," and is subjected to extra scrutiny upon their next attempted login.


This is where Poze comes in. Using an existing photo as reference, the site asks Conlon (the user) to complete a simple task to prove that he's a human. Beyond being a CAPTCHA, this new photo is compared against the reference photo using Amazon Rekognition to determine that it is indeed the same user (haircuts and glasses notwithstanding). Having passed this first check, Poze then determines if the user successfully completed the task within the required timeframe. The combination of randomly generated prompts and a timeboxed interaction window works to ensure that the picture can't be spoofed or deepfaked.


Poze API Features

  • • Randomly generated and timeboxed micro-activities allow users to prove that they're the same human without submitting government IDs to monolithic corporations.
  • • API microservice design lends itself to further implementations, including AR and miexed reality tasks.
  • • Honestly? It's just plain fun. When was the last time you enjoyed filling out a CAPTCHA?
